# 📱 Деплой в Google Play — пошаговое руководство

## Что мы делаем

Заворачиваем веб-приложение в **TWA (Trusted Web Activity)** — это нативный Android-контейнер, который запускает app.каталог-сочи.рф в полноэкранном режиме без адресной строки. Пользователь видит обычное приложение, не отличимое от нативного.

Преимущества: один источник правды (веб-код), мгновенные обновления без релиза в Play, поддержка push, оффлайн через Service Worker.

---

## Шаг 1 — Зарегистрируй Google Play Console

🔗 https://play.google.com/console

- Аккаунт «Личный» (Individual)
- Платёж $25 одноразово (~2400₽), Visa/Mastercard РФ работает не всегда — могут понадобиться альтернативы (Тинькофф валютная, Союз, виртуальные карты)
- Заполни профиль разработчика (имя, email, телефон)

⏱ 10-15 минут.

---

## Шаг 2 — Сгенерируй AAB через PWABuilder

🔗 https://www.pwabuilder.com

1. В поле URL вставь: `https://app.каталог-сочи.рф`
2. Нажми **Start**
3. PWABuilder проанализирует приложение → должен дать оценку 90+
4. Нажми **Package for Stores**
5. Выбери **Android**
6. **App package ID**: `ru.kataloghi.crmagenta` (любой уникальный, формат как у DNS наоборот)
7. **App name**: `CRM Агента`
8. **Launch URL**: `/`
9. Оставь остальное по умолчанию (24-bit signed key)
10. **Generate**
11. Скачай **`.zip`** — внутри будут:
    - `app-release-signed.aab` — то что заливать в Play
    - `assetlinks.json` — нужно положить на хостинг (см. Шаг 3)
    - `signing-key.keystore` + пароли — **сохрани навсегда**, без него не сможешь обновлять приложение!

⏱ 5 минут.

---

## Шаг 3 — Подтверди связь домена с приложением

Это обязательно для TWA — без этого Android будет показывать адресную строку.

1. Открой полученный `assetlinks.json` — там SHA256-отпечаток твоего ключа
2. Помести этот файл на сайт по пути: `https://app.каталог-сочи.рф/.well-known/assetlinks.json`

**Как:**
- В GitHub-репо `Catalog-sochi` создай папку `.well-known` (через UI: New file → имя `.well-known/assetlinks.json`)
- Вставь содержимое JSON из PWABuilder
- Commit

Проверь что доступно по адресу `https://app.каталог-сочи.рф/.well-known/assetlinks.json` — должен открыться JSON.

⏱ 5 минут.

## Шаг 5 — Залей в Play Console

1. Play Console → **Create app**
2. App name: `CRM Агента`
3. Default language: Russian
4. App or game: App
5. Free or Paid: Free
6. **Production → Create new release**
7. Загрузи `app-release-signed.aab`
8. Заполни Release notes (что нового)
9. **Save** → Review release → **Start rollout to production**

Google проверяет 1-7 дней. После одобрения приложение появится в Play.

---

## Шаг 6 — Push-уведомления (опционально, после публикации)

Browser Notification API уже работает в TWA — пользователи могут включить в Профиле → «Включить уведомления».

Для нативных push (через FCM) понадобится:
- Firebase проект
- Service Worker с FCM SDK
- Бэкенд для отправки

На MVP этого не нужно — браузерные push покрывают 95% сценариев.

---

## Что после публикации

Когда нужно обновить приложение:
1. Меняешь код → деплоишь в GitHub Pages
2. Через 1-2 минуты обновления автоматически прилетают всем (Service Worker подхватывает)
3. **Релиз в Play делать НЕ нужно** — это магия TWA

Только при изменении package ID, ключей или нативных функций — собирается новая AAB.
