# Переход на Supabase — пошагово

Делается за один вечер. После этого данные синхронизируются между всеми устройствами автоматически.

## Шаг 1 — Создать проект (5 минут)

1. Откройте supabase.com → **Sign up** (через Google)
2. **New project** → название `catalog-sochi` → пароль базы данных (запишите!) → регион `Frankfurt (eu-central-1)` (ближе к РФ) → **Create**
3. Ждите 1–2 минуты пока создаётся

## Шаг 2 — Залить схему (1 минута)

1. В левом меню **SQL Editor** → **New query**
2. Откройте файл `supabase-schema.sql` из вашей папки → скопируйте всё содержимое
3. Вставьте в SQL Editor → **Run** (внизу справа)
4. Должно показать «Success. No rows returned» — это нормально

## Шаг 3 — Создать первого пользователя (вас)

1. Слева **Authentication** → **Users** → **Add user** → **Create new user**
2. Email: ваш email, Password: придумайте, **Auto Confirm User: ✓** → **Create**
3. Скопируйте появившийся User UID
4. Вернитесь в SQL Editor → **New query**, вставьте и Run:

```sql
insert into agents (auth_user_id, name, role, phone, city)
values ('ВСТАВЬТЕ_СЮДА_UID', 'Алёна', 'Директор', '+7...', 'Сочи');
```

## Шаг 4 — Создать bucket для фотографий

1. Слева **Storage** → **New bucket**
2. Name: `property-photos`, **Public bucket: ✓** → **Create**

## Шаг 5 — Получить ключи

1. Слева **Project Settings** (шестерёнка) → **API**
2. Запишите две строки:
   - **Project URL** (например `https://abcde.supabase.co`)
   - **anon public** ключ (длинная строка `eyJ...`)

Эти две строки — отправьте мне следующим сообщением, я вставлю их в `index.html` и перепишу сохранение/чтение данных на Supabase вместо localStorage. После этого:

- Данные одинаковы на телефоне и компьютере
- При установке на телефон других агентов — они логинятся по email/паролю
- Фото хранятся в облаке, не съедают память телефона
- Все правила доступа (агент видит только своих клиентов, директор — всех) уже зашиты в БД на уровне Row Level Security

## Что после Supabase

После рабочей синхронизации останется только:

1. **Регистрация по телефону** — Supabase Auth поддерживает SMS через Twilio (платно ~$1/мес)
2. **Telegram-вход** — через Telegram Login Widget
3. **Нативные приложения** для App Store/Play Store — оборачиваем через Capacitor (1–2 дня), код не меняется
4. **Юридические документы** — Политика конфиденциальности и Пользовательское соглашение (могу сгенерировать шаблоны)

Готово к публикации в магазинах будем после этих шагов.
